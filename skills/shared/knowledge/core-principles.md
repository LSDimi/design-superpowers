# Core Design Principles (L1 — Always Loaded)

> Compact reference loaded into every sub-agent. Total budget: ~2K tokens.

## How to use

When making any design decision, recall these principles. They are the floor, not the ceiling. Domain-specific knowledge (L2) extends these. Severity: Critical = never violate, High = strong default, Medium = context-dependent.

## Principles

### 1. Innovative
**Do:** Use technology to improve function; seek new possibilities.
**Don't:** Repeat existing forms without new utility; treat innovation as an end.
**Severity:** High

### 2. Useful
**Do:** Prioritize functional, psychological, and aesthetic utility.
**Don't:** Include features that don't serve the primary function.
**Severity:** Critical

### 3. Understandable
**Do:** Make structure self-explanatory; let function speak to the user.
**Don't:** Require manuals for basic use; impose high cognitive load.
**Severity:** Critical

### 4. Honest
**Do:** Be transparent about capabilities and performance.
**Don't:** Manipulate with unfulfillable promises or false representations.
**Severity:** Critical

### 5. Minimal
**Do:** Concentrate on essentials only; strive for purity.
**Don't:** Burden the design with non-essentials or unnecessary complexity.
**Severity:** Critical

### 6. Aesthetic
**Do:** Create harmonious, well-executed objects where beauty serves utility.
**Don't:** Treat aesthetics as secondary or purely decorative.
**Severity:** High

### 7. Durable
**Do:** Design for longevity; avoid fashionable choices that date quickly.
**Don't:** Follow short-lived trends that lead to throwaway design.
**Severity:** High

### 8. Detailed
**Do:** Ensure every detail is functional and precise; leave nothing to chance.
**Don't:** Leave loose ends or allow sloppy finishes.
**Severity:** High

### 9. Unobtrusive
**Do:** Keep design neutral and restrained; leave room for user expression.
**Don't:** Over-design products; overwhelm users with loud elements.
**Severity:** Medium

### 10. Equitable Use
**Do:** Provide the same means of use for all users; make it appealing to all.
**Don't:** Segregate or stigmatize any users.
**Severity:** Critical

### 11. Simple and Intuitive Use
**Do:** Eliminate unnecessary complexity; match user expectations.
**Don't:** Require high literacy or cognitive load for basic navigation.
**Severity:** Critical

### 12. Perceptible Information
**Do:** Use multiple modes (visual, verbal) and high contrast to communicate.
**Don't:** Rely on color or a single sense alone to convey critical meaning.
**Severity:** Critical

### 13. Tolerance for Error
**Do:** Provide warnings, fail-safes, and easy undo options.
**Don't:** Design tasks that lead to irreversible or catastrophic errors.
**Severity:** Critical

### 14. Flexibility in Use
**Do:** Accommodate a range of preferences and abilities; offer method choice.
**Don't:** Force a single interaction path or assume uniform user speed.
**Severity:** High

### 15. Low Physical Effort
**Do:** Allow neutral body position; minimize repetitive actions.
**Don't:** Require excessive force or sustained physical effort.
**Severity:** Medium

### 16. Business Strategy Alignment
**Do:** Align design goals with business KPIs; identify sustainable outcomes.
**Don't:** Design in a vacuum ignoring business goals.
**Severity:** High

### 17. Value Innovation
**Do:** Pursue differentiation and efficiency simultaneously; create new value.
**Don't:** Add features purely to copy competitors.
**Severity:** High

### 18. Validated User Research
**Do:** Use Build-Measure-Learn; build MVPs to test assumptions.
**Don't:** Assume you know what users want without direct input.
**Severity:** Critical

### 19. Killer UX Design
**Do:** Create frictionless flows; focus on the key user journey.
**Don't:** Clutter interfaces with features that obscure core tasks.
**Severity:** Critical

### 20. Decision Making / Iterating
**Do:** Use prototypes to fail fast; include stakeholders in evaluation.
**Don't:** Be paralyzed by indecision; use arbitrary selection standards.
**Severity:** High

### 21. Problem Definition / Validation
**Do:** Clearly define the user problem before building; validate solutions.
**Don't:** Solve problems that don't exist; skip validation before implementation.
**Severity:** Critical

### 22. Thinking Systemically
**Do:** Map dependencies across touchpoints; design for ecosystem coherence.
**Don't:** Focus on isolated interfaces; ignore connected systems.
**Severity:** High

### 23. Simple (Sticky Ideas)
**Do:** Strip to the essential message; use concrete analogies.
**Don't:** Bury the lead; overcomplicate with needless data.
**Severity:** High

### 24. Credible
**Do:** Use statistics with context; provide specific, vivid supporting details.
**Don't:** Make unsubstantiated claims; use statistics without context.
**Severity:** High

### 25. Emotional Connection
**Do:** Appeal to self-interest or identity; use human connections.
**Don't:** Stay purely analytical; ignore the emotional dimension.
**Severity:** Medium

### 26. Nail the Basics
**Do:** Ensure core features work perfectly every time; prioritize reliability.
**Don't:** Prioritize flashy features over system stability and utility.
**Severity:** Critical

### 27. Build Confidence
**Do:** Provide clear feedback, transparent data, and safety markers.
**Don't:** Hide critical information; use dark patterns.
**Severity:** Critical

### 28. Purposeful Motion
**Do:** Use motion to show hierarchy, relationships, and direct attention.
**Don't:** Use motion without clear functional intent.
**Severity:** High

### 29. Seamless Transitions
**Do:** Coordinate simultaneous changes; ensure high frame rates.
**Don't:** Allow stuttering or janky transitions.
**Severity:** High

### 30. Functional Motion
**Do:** Use motion to guide user focus to new tasks.
**Don't:** Add decorative animations that slow the user down.
**Severity:** High

### 31. Customer Collaboration
**Do:** Use participatory design; involve users throughout.
**Don't:** Rely solely on rigid initial requirements or assumptions.
**Severity:** High
