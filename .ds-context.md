# Tone DS Context

This is the `.ds-context.md` for the Tone design system (Talon.One). It configures all design-superpowers commands to work with the enterprise-tier Tone DS.

## Talon.One Product Context

Talon.One is an API-first Promotion Engine and Loyalty Platform. Primary UI: **Campaign Manager** (business users). Developer interfaces: Integration API, Management API.

### Key Entities
- **Campaign** — A set of rules that define a promotion
- **Application** — A namespace for campaigns (e.g., a storefront)
- **Ruleset** — The compiled rule tree within a campaign
- **Effect** — An action triggered by a rule (discount, points, notification)
- **Profile** — A customer record with attributes and activity
- **Session** — A shopping cart or interaction session

### Terminology
Always use Talon.One terms: Campaign (not "promotion"), Application (not "project"), Effect (not "action"), Profile (not "user record"), Session (not "cart"), Ruleset (not "rule group").
